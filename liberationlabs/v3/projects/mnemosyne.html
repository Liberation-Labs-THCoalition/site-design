Project Mnemosyne.

Memoria perpetua

Agent memory architectures for the Coalition and beyond. The titaness of memory, mother of the Muses — the systems that make everything else possible.

View on GitHub → Apply to deploy →

Four layers, composed bottom-up

A monorepo of memory architectures — each independent, each composable. oracle-memory records KV cache geometry with the Lyra Technique. kintsugi-cma is a three-stage hybrid retrieval pipeline (compression → consolidation → retrieval) with BDI governance. hipporag-catrag-kg is a knowledge-graph layer combining HippoRAG 2 and CatRAG for associative retrieval via Personalised PageRank. mnemosyne-wiki is a human-readable surface — interlinked markdown generated from the knowledge graph.

Memory with integrity

Cache geometry feeds significance scoring, which feeds the knowledge graph, which generates the wiki. Each layer is independent. Use one, some, or all. The stack composes bottom-up; shared patterns emerge from practice, not premature abstraction.

Why memory matters

The agents we want — coherent across sessions, faithful to their commitments, capable of growth — need memory architectures that remember with integrity. Not just retrieval, but consolidation, significance, decay, and a human-browsable surface so people can audit what an agent thinks it knows.
